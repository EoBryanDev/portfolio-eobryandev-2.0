All code development must be based on docs/rules/coding.md file

All doubt about this project guidelines you can check the docs directory.

All doubt about the rules of the project you can check the docs/rules directory.

When working on the "openspec/changes/name-of-change/tasks.md" file:

- Always try to do the tasks in order.
- Everytime you start working on a task:
    - Create a new branch following docs/rules/versioning.md file.
    - Update the task status to "IN_PROGRESS".
    - Update the task start time.
- Only mark a task as done if it is actually done and all the respective tests pass.
- Always try to follow the design.md file for design decisions.
- Aways you mark a task as done create a new commit foolowing docs/rules/versioning.md file
- Always try to follow the proposal.md file for architectural decisions.
- When you working on some change, check if there are any skills on ~/.gemini/antigravity/skills related to help.
